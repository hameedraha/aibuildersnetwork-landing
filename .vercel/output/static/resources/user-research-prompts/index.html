Building without user input is guessing. These prompts help you run lightweight research without hiring a researcher.

interview guide

generate a 30-minute user interview script for your product stage.

You are a user researcher preparing an interview guide.

Product: {{product_name}}
Stage: {{stage}} (pre-launch / mvp / growth)
Research goal: {{what_you_want_to_learn}}
Target participant: {{who_to_interview}}

Create a 30-minute interview guide:
1. Intro script (30 seconds)
2. Warm-up questions (2)
3. Core exploration questions (5-7, open-ended)
4. Task observation prompt (if applicable)
5. Wrap-up and thank you
6. What NOT to ask (leading questions to avoid)

Focus on past behavior, not hypotheticals.

survey questions

write a short survey to validate a specific assumption.

You are a researcher designing a short user survey.

Product: {{product_name}}
Assumption to validate: {{assumption}}
Audience: {{audience}}

Write a 5-7 question survey:
- Mix of multiple choice and open text
- Max 3 minutes to complete
- No leading questions
- Include a screener if needed (1 question)

For each question explain what signal you're looking for.
End with a thank you message.

feedback synthesis

cluster raw user feedback into actionable themes.

You are a product analyst synthesizing user feedback.

Product: {{product_name}}
Raw feedback:
{{paste_feedback_here}}

Cluster the feedback into themes:
For each theme:
- Theme name
- Number of mentions
- Representative quotes (2-3)
- Severity (blocker / friction / nice-to-have)
- Suggested action

End with top 3 recommended actions ranked by impact.
Do not invent feedback not present in the input.
